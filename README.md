# Spark Bot

ESP-SparkBot 复刻与二次开发项目。

这个仓库不是单纯照搬官方教程，而是把 SparkBot 当成一个可以长期迭代的桌面 AI 机器人平台来做：先复刻跑通，再逐步加入自定义固件、AI 对话、桌面控制台、开发工作流提醒等能力。

## 项目目标

- 复刻 ESP-SparkBot 硬件并完成首台装机
- 跑通官方固件与基础功能
- 梳理可替换 BOM、焊接顺序、排障流程
- 定制固件能力：屏幕、音频、摄像头、履带运动、Wi-Fi、语音交互
- 接入大模型服务，例如 DeepSeek、OpenAI、Qwen 等
- 发展为个人桌面 AI 助手，连接开发工具、GitHub、天气、日程、骑行计划等

## 参考资料

- 开源复刻教程：https://sparkbot.com.cn
- 立创开源项目：https://oshwhub.com/esp-college/esp-sparkbot

## 仓库结构

```text
.
├── apps/           # 桌面控制台、调试工具、未来的 Web/Electron 应用
├── docs/           # 文档、路线图、BOM、装配与排障记录
├── firmware/       # ESP-IDF 固件、示例、二次开发代码
├── hardware/       # PCB、外壳、3D 打印、接线、装配相关资料
└── scripts/        # 辅助脚本，例如烧录、配置、资源转换
```

## 阶段路线

1. **P0：资料收集**  
   确认官方资料、复刻教程、BOM、PCB、外壳、固件源码。

2. **P1：硬件复刻**  
   PCB 打样、元件采购、焊接、上电检查、屏幕/音频/摄像头验证。

3. **P2：官方固件跑通**  
   搭建 ESP-IDF 环境，编译烧录官方示例，逐个验证功能。

4. **P3：自定义固件**  
   抽象显示、语音、运动、网络、配置系统，开始做自己的功能。

5. **P4：AI 桌面助手**  
   接入大模型、语音识别、TTS、Agent 状态提醒、GitHub/日程/天气等插件。

## 开发原则

- 每一步都留下记录：买了什么、怎么焊、哪里失败、怎么修好。
- 优先让硬件稳定，再做复杂软件。
- 固件先小步验证，不一口气改大工程。
- 保留官方可运行版本，再逐步 fork 出自己的系统。
