# Roadmap

## P0：资料收集与项目拆解

目标：搞清楚 SparkBot 的硬件、固件、外壳、BOM、烧录方式和风险点。

- [ ] 阅读 sparkbot.com.cn 复刻教程
- [ ] 阅读立创开源项目页面
- [ ] 找到官方/复刻固件源码
- [ ] 整理 BOM
- [ ] 标记容易买错的器件
- [ ] 标记需要 3D 打印的外壳件
- [ ] 标记需要提前准备的工具

验收标准：能明确知道要买什么、打什么板、打印什么外壳、第一步怎么上电。

## P1：硬件复刻

目标：完成第一台 SparkBot 的硬件装配。

- [ ] PCB 下单
- [ ] 元器件采购
- [ ] 外壳打印
- [ ] 电源部分焊接与检查
- [ ] ESP32-S3 主控部分焊接与检查
- [ ] 屏幕、摄像头、音频、麦克风验证
- [ ] 履带、电机、驱动验证
- [ ] 整机装配

验收标准：不开烟、不发烫、能稳定供电、能进入烧录模式。

## P2：官方固件跑通

目标：先不要大改，原样跑起来。

- [ ] 安装 ESP-IDF
- [ ] 拉取官方/复刻源码
- [ ] 编译第一个最小固件
- [ ] 烧录并查看串口日志
- [ ] 跑通屏幕显示
- [ ] 跑通音频播放
- [ ] 跑通摄像头
- [ ] 跑通 Wi-Fi
- [ ] 跑通运动控制

验收标准：官方核心功能至少跑通 3 个以上，确认硬件没有大问题。

## P3：自定义固件

目标：开始把 SparkBot 改成自己的机器人。

- [ ] 梳理源码结构
- [ ] 抽象配置系统
- [ ] 替换/新增大模型 API 配置
- [ ] 加入本地调试日志
- [ ] 定制屏幕 UI
- [ ] 增加基础状态页面
- [ ] 增加简单插件机制

验收标准：不改硬编码也能切换模型/API，机器人能显示自己的状态。

## P4：桌面 AI 助手

目标：让 SparkBot 与日常开发工作流联动。

- [ ] 设计桌面控制台
- [ ] WebSocket/MQTT 通信方案
- [ ] GitHub 通知插件
- [ ] AI Coding Agent 状态提醒
- [ ] 天气/日程/骑行提醒插件
- [ ] 机器人主动播报能力

验收标准：SparkBot 能主动提醒某个真实任务状态，而不只是被动聊天。
