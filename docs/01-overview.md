# 01. 项目总览

## 我们要做的不是一次性复刻

SparkBot 可以被当成三层项目来看：

1. **硬件复刻项目**：把官方 ESP-SparkBot 做出来。
2. **固件学习项目**：理解 ESP32-S3 上的屏幕、音频、摄像头、触摸、IMU、网络、AI 功能。
3. **桌面 AI 助手项目**：在官方基础上做自己的定制固件和桌面端能力。

所以这个仓库的目标不是“买齐零件然后碰运气”，而是沉淀一份从零复刻、排障、开发、扩展的完整笔记。

## 官方项目关键信息

- ESP-SparkBot 基于 ESP32-S3。
- 功能覆盖语音交互、图像识别、遥控操作、多媒体娱乐、AI 对话、实时视频传输、USB 投屏等。
- 官方开源协议标注为 GPL 3.0。
- 官方页面显示创建时间为 2024-10-15，更新时间为 2025-03-12。
- 固件源码托管在 Gitee：`esp-friends/esp_sparkbot`。

## 版本判断

优先按 **V1.2** 复刻。

原因：

- V1.0 主体头部没有断电开关。
- V1.1 新增头部断电开关、修改结构、履带底盘 PCB 改为一块板、修复触摸 FPC 问题。
- V1.2 在 V1.1 基础上优化 ES8311 部分电路，用于解决大模型对话时喇叭播放声音卡顿问题。

## 第一台机器的策略

### 不追求一步到位

第一台机器优先目标：

1. 不烧板。
2. 能烧录。
3. 屏幕亮。
4. 串口日志正常。
5. Wi-Fi、摄像头、音频逐个验证。
6. 最后再做履带小车和 AI 对话。

### 不急着魔改

先保留一个“官方可运行状态”，再开分支做自己的 SparkBot OS。

原因很简单：硬件阶段一旦出现问题，需要一个稳定固件做对照。否则黑屏、无声音、重启时，很难判断是硬件错、固件错还是配置错。

## 当前最高风险

| 风险 | 原因 | 处理策略 |
| --- | --- | --- |
| 头部 PCB 插接方向 | 官方明确提示接反极大概率烧板 | 装配前拍照标记方向 |
| 电池供电不稳 | 官方已知问题：摄像头和 Wi-Fi 同时工作功耗高，低电压会重启 | 首次调试优先 USB 供电，电池后测 |
| BMI270 买错 | 官方提示淘宝可能买到 BMI220 | 优先嘉立创采购，来料核对丝印/型号 |
| 结构装配困难 | 主体 3 块 PCB 插接，还有触摸 FPC | 先裸板调试，再装壳 |
| 屏幕不亮误判 | 复刻教程提示先刷机屏幕才会点亮 | 不把未刷机黑屏当故障 |
